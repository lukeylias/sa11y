sa11y

Easy-to-follow steps for beginners creating accessible websites.

Fundamentals

Use semantic HTML

Semantic HTML helps screenreader users to understand the content's structure and gives it meaning. Ask yourself "What is it?" when writing HTML. Use a logical headline order and structuring elements such as header, footer, ul, p...

Learn more →

Provide text for non-text content

Every content not represented by text is invisible for screen readers. Provide concise descriptions in alt attributes for all content images and text alternatives for charts etc. CAPTCHAs also need to be accessible.

Learn more →

Add labels to form elements

"Form input elements need linked <label>s for screenreaders access. Example: <label for='search'>Search</label><input id='search'> The placeholder attribute is not an alternative."

Learn more →

Show where your :focus Is

Navigating by keyboard needs the currently focused element to be visibly highlighted. Interactive elements should get focused in a useful order and be focusable by default. E.g. take a < button > instead of a ‹span>

Understandable without color

Colours should not be necessary to understand the content and its current state. For example don’t highlight errors just with red text. Add a subsidiary icon.

Dynamic sites such as single page applications tend to get complicated when it comes to accessibility. Semantic HTML should always be considered before using WAI-ARIA.
